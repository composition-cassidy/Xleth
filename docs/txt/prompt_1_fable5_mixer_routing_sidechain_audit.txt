TASK SUMMARY
Perform a read-only architecture audit for XLETH mixer bus routing and sidechain support, then produce a concrete phased implementation plan that will become the source-of-truth for later implementation prompts.

Context:
- Project: XLETH.
- Stack: Windows-only C++ engine using JUCE + FFmpeg + OpenGL, Node-API bridge, Electron + React renderer.
- The feature needed is mixer routing plus sidechain support.
- Mixer bus routing means one or more audio tracks can output into another mixer/audio track as an audible bus.
- Sidechain routing means a source track can feed a silent key/control signal into a target effect or FX Graph sidechain input without becoming audible through that target.
- The key user problem to avoid: if Kick is output-routed to Drum Bus, Kick must not also keep going directly to Master by default. That would cause double-monitoring and louder/incorrect playback.
- Reference docs to read first:
  - docs/txt/xleth_mixer_routing_feature_brief.txt
  - docs/txt/juce_audio_graph_sidechain_reference.txt

Relevant areas to inspect:
- Engine audio graph/routing:
  - engine/src/audio
  - existing AudioGraph, MixEngine, EffectChainManager, track processing, mixer track structures, mute/solo, latency/PDC, offline render/export paths.
- Bridge/API:
  - Node-API addon files
  - Electron IPC in ui/main.js
  - preload APIs in ui/preload.js
- Renderer/UI/state:
  - mixer components/stores
  - effect chain store
  - FX Graph graphState helpers and panel components
  - project save/load serialization and migrations
  - undo/redo command patterns
- Tests:
  - existing engine audio tests
  - existing renderer/store tests
  - export/offline render tests if present.

Hard constraints:

- Do not hardcode production UI colors if you discuss UI implementation later.

Routing semantics to preserve:
- Default normal track output should remain Master unless changed.
- Output routing to a bus means the source's main audible output goes to the bus instead of directly to Master.
- Parallel sends are separate from output routing. Do not conflate them.
- Sidechain routes are silent detector/control routes. They do not contribute directly to audible output.
- FX Graph sidechain should be represented by a protected Sidechain Input / Input Activator node that cannot be deleted.
