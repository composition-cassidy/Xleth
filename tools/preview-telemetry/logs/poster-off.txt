Notes for the developer reading this report:
  • The Sample Selector and the imported-video popup BOTH use Chromium <video>
    elements served via http://127.0.0.1 — they are completely independent of the
    GridCompositor / shared-memory / WebGL pipeline. They working tells you nothing
    about the main preview path.
  • The "Hardware/Software" video mode setting only affects DECODE/ENCODE; it does
    NOT bypass the D3D11 GridCompositor or the WebGL canvas. There is no full CPU
    fallback for the main preview.
  • If section 4 shows readbackValid > 0 and section 5 shows framesReceived === 0,
    the failure is between the engine writing to the file mapping and the renderer
    reading from it — check shm_helper.node load and FRAME_SHM_NAME match.
  • All GPUs start on FastImmediate (blocking Map, no DO_NOT_WAIT, ~0-2ms on NVIDIA).
    If avgReadbackMs>8, fatalInvalids>3, or yield<25% in a 60-frame window,
    the engine auto-switches to AsyncQueued (DO_NOT_WAIT ring, 5 slots).
    D3D11_MAP_FLAG_DO_NOT_WAIT = 0x00100000 (confirmed in readbackMapFlags log).
  • readbackInvalid(fatal)>0 = descriptor/device issue, not GPU latency.
  • readbackNotReady in AsyncQueued = GPU behind tick rate; previous frame stays visible.
  • droppedPendingFrames in AsyncQueued is non-fatal: oldest frame dropped, preview continues.
